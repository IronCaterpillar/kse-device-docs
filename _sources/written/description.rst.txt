Введение
====================
Данное руководство содержит краткую инструкцию по работе с ПО устройства, а так же объясняет
принципы его функционирования, внутреннюю организацию и описание работы веб-интерфейса настройки.

Общие принципы работы
===================

ПО устройства подразделяется на *системное* *ПО* и *приложение управления электрообогревом*. Системное ПО отвечает за загрузку устройства, работу файловой системы,
контроль состояния, индикацию, обработку каналов ввода-вывода, работу сетевых интерфейсов. Приложение отвечает за задачу регулирования,
контроля тока, защитное отключение в случае превышения допустимых пределов токов/температуры. Настройка устройства ведётся 
с помощью встроенного в устройство веб-интерфейса.

Системное ПО
---------------
Системное ПО устанавливается в процессе производства. При необходимости устройство проходит заводскую калибровку каналов аналогового ввода.
Системное ПО может быть при необходимости обновлено для актуализации, либо для получения дополнительных возможностей. Основой системного ПО
является словарь объектов, где отображаются текущие данные, параметры и настройки устройства. Подробное содержание словаря находится в соответствующем разделе. 
Словарь объектов является так же источником информации для отправки по интерфейсам связи. Для этого служат файлы настройки отображения. 
Каждому протоколу передачи данных соответствует свой файл/файлы настройки отображения, например, для протокола *Modbus* служит файл 
*modbus_mappings.cfg*. Описание процесса настройки отображения и настройку по умолчанию можно найти в соответствующих разделах.

Приложение управления электрообогревом
--------------------------------------
Приложение так же устанавливается в процессе производства и может быть заменено независимо от системного ПО. Осуществляет управление линией электрообогрева, контроль
параметров и защитное отключение нагрузки.
